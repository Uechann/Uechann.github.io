# PRD-14: 관리자 페이지 v2

> 상태: 🔴 미구현  
> 우선순위: P0  
> 의존: PRD-13 (카테고리 트리)

---

## 배경

현재 관리자의 기능(글 CRUD, 카테고리 관리)을 유지하되, 세 가지 큰 영역으로 재구성한다:
1. **내 정보 관리** — 프로필 사진, 링크, 소개글
2. **카테고리 관리** — Composite 트리 구조
3. **글 관리** — 작성/수정/삭제

## 2. 카테고리 관리 (Composite 트리)

```
┌──────────────────────────────────────────────────┐
│  ⚙️ 카테고리 관리                                  │
│  ──────────────────────────────────────────────  │
│                                                  │
│  ▼ 📚 개념학습                       [+ 하위 추가]│
│    ▼ ☕ Java              (5 글)     [이름변경][삭제]│
│      • 트랜잭션 전파 이해하기                     │
│      • JVM 메모리 구조                            │
│    ▶ 🌱 Spring            (3 글)     [이름변경][삭제]│
│  ▶ 🏫 우아한테크코스                  [+ 하위 추가]│
│  [+ 최상위 폴더 추가]                             │
└──────────────────────────────────────────────────┘
```

### 폴더 추가 폼
```
┌────────────────────────────────────┐
│  새 폴더 추가 (상위: 개념학습)       │
│  키 (영문, 불변)  [my-category___] │
│  이름             [내 카테고리____] │
│  아이콘           [🗂️_____________] │
│  색상             [color picker]   │
│  [저장]  [취소]                    │
└────────────────────────────────────┘
```

### 제약사항
- **삭제**: 해당 폴더와 모든 하위에 글이 0개일 때만 가능
- **이름변경**: label만 변경, key(경로)는 불변
- **이동**: 미지원 (URL 변경으로 SEO 영향)
- **글 목록**: 최종 폴더(children=[])에서 해당 카테고리 글 목록 표시 (읽기 전용)

## 3. 글 관리

현재 사이드바 트리에서 글을 클릭해 편집하는 방식 유지 + 개선.

### 3-1. 새 글 작성

카테고리 트리에서 `+ 새 글 작성` 클릭 → 에디터 오픈.

```
┌──────────────────────────────────────────────────────────┐
│  ☕ Java  › 새 글 작성                                    │
│                                                          │
│  제목    [________________________________]              │
│  슬러그  [auto-from-title_______________]                │
│  날짜    [2026-06-15]                                    │
│  공개 여부  ○ 초안  ● 발행                               │
│                                                          │
│  카테고리  [☕ Java ✓] [🌱 Spring] ...                   │
│  태그      [Java] [Spring] [+ 태그 입력]                 │
│  소개글    [___________________________________]         │
│  썸네일    [이미지 URL 입력]                             │
│                                                          │
│  ┌───────────────────────────────────────────────────┐  │
│  │  EasyMDE 에디터                                    │  │
│  └───────────────────────────────────────────────────┘  │
│                                                          │
│  [초안 저장]  [발행]  [삭제]                             │
└──────────────────────────────────────────────────────────┘
```

### 3-2. 글 수정

사이드바 트리에서 글 클릭 → 동일 에디터 오픈, 기존 내용 로드.

### 3-3. 글 삭제

에디터 하단 `[삭제]` 버튼 → 확인 다이얼로그 → GitHub API로 파일 삭제.

## 4. 추가 제안 기능

### 4-1. 대시보드 (Welcome 화면)
```
┌────────────────────────────────────────────┐
│  안녕하세요! 👋                              │
│                                            │
│  전체 글   카테고리   발행   초안            │
│    23개      10개    18개    5개            │
│                                            │
│  최근 수정한 글                             │
│  • 2026-06-15  트랜잭션 전파 이해하기       │
│  • 2026-06-12  JVM 메모리 구조             │
└────────────────────────────────────────────┘
```

### 4-2. 이미지 업로드 보조
- GitHub API로 `static/img/` 에 이미지 파일 업로드
- 업로드 후 URL 자동 복사 → 에디터에 붙여넣기
- 지원 형식: JPEG, PNG, WebP, GIF

### 4-3. 글 검색 (관리자 내부)
- 사이드바 상단 검색창
- 현재 로드된 글 제목/슬러그로 필터링

### 4-4. 발행/초안 상태 일괄 관리
- 카테고리별로 초안 글 목록 한눈에 확인
- 필터: 전체 / 발행 / 초안

## 기술 결정

- 관리자는 현재와 동일하게 순수 HTML/CSS/JS SPA 유지
- GitHub REST API로 파일 CRUD (PAT 인증)
- 카테고리 트리는 재귀 렌더링 (JavaScript)
- EasyMDE 마크다운 에디터 유지
