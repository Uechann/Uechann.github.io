# PRD-17: 블로그 매력도·재미 요소 추가

> 상태: 🔴 미구현
> 우선순위: P2 (콘텐츠 동기화 안정화 이후)
> 의존: 없음 (대부분 독립적, 점진 도입 가능)

---

## 배경

현재 블로그는 글·카테고리·댓글(giscus)·다크모드 등 기본기는 갖췄지만,
**처음 방문한 사람의 흥미를 끌 요소**와 **읽는 재미**가 부족하다.
방문자가 "오 괜찮네, 더 둘러볼까?" 하게 만드는 장치를 추가한다.

핵심 질문 3가지로 기능을 골랐다.
1. **첫인상** — 들어오자마자 "이 블로그 뭔가 있다"는 느낌을 주는가?
2. **읽는 재미** — 글 한 편을 끝까지 읽게 만드는가?
3. **둘러보게** — 한 글에서 다음 글로 자연스럽게 넘어가는가?

---

## 목표

- 홈 화면 체류 시간·재방문·글 탐색(페이지뷰/세션) 증가
- 글 읽기 완독률 향상
- "개발자가 정성껏 만든 블로그"라는 인상

### 비목표
- 광고·트래킹 과다(프라이버시 침해) 도입
- 무거운 SPA 전환 (Hugo 정적 사이트 성격 유지)

## 추천 우선순위 (1차 도입 = "최대 효과 / 최소 공수")

> 아래 7개만 해도 체감 매력도가 크게 오른다. 모두 S 공수, JS/CSS·Hugo 파셜로 해결.

1. **B3 코드 복사 버튼** — 개발 블로그 기본기
2. **B1 읽기 진행 바** + **B2 읽기 시간** — 완독 유도
3. **C1 관련 글** + **C2 이전/다음 글** — 다음 글로 유도
4. **A1 홈 히어로 + 통계** — 첫인상
5. **A3 포스팅 잔디** — "와" 포인트 (M이지만 임팩트 최고)

---

## 구현 방법 (이 레포 기준)

- **B1/B3/B6 (클라이언트 동작):** `static/js/blog.js`에 스크롤·복사·맨위로 로직 추가, `static/css/blog.css`에 스타일. 추가 의존성 0.
- **B2 읽기 시간:** Hugo `.WordCount`는 CJK에 부정확 → `len`(룬 수) 기반 커스텀 계산을 `layouts/_default/single.html`에서 처리 (한글 약 500자/분 가정).
- **C1 관련 글:** `hugo.yaml`에 `related` 설정(키: tags, categories) 후 `single.html`에서 `.Site.RegularPages.Related .` 렌더.
- **C2 이전/다음:** `.PrevInSection`/`.NextInSection`.
- **A1 통계:** `len .Site.RegularPages`, `len .Site.Taxonomies.tags` 등으로 `layouts/index.html`에서 계산.
- **A3 잔디:** 빌드 시 글 날짜를 `data`로 모으거나 파셜에서 `.Site.RegularPages`를 날짜별 집계 → CSS grid 52×7 셀 렌더. JS 불필요(정적 생성) 또는 경량 JS.
- **C4 검색 강화:** Hugo로 `index.json`(제목·요약·본문·URL) output 생성 → Fuse.js로 클라이언트 검색.
- **D2 조회수:** GoatCounter 스크립트를 `head.html`에 1줄, 글별 카운트 API로 표시.

## 요구사항

### R1 — 읽기 경험 (P0 묶음)
- 코드 복사 버튼, 읽기 진행 바, 예상 읽기 시간

### R2 — 탐색 유도 (P0/P1)
- 관련 글 추천, 이전/다음 글

### R3 — 홈 첫인상 (P0)
- 히어로 + 블로그 통계(글/카테고리/태그 수)

### R4 — 포스팅 잔디 (P0, 임팩트)
- 1년치 발행 히트맵, 셀 hover 시 날짜·글 수 툴팁

### R5 — 부가 재미 (P1~P2, 선택)
- 이미지 라이트박스, 맨위로, TOC 스크롤스파이, 조회수, 태그 클라우드, 재밌는 404, 방명록, 이스터에그
