# PRD-12: 블로그 전면 개편 — 커스텀 테마

> 상태: 🔴 미구현  
> 우선순위: P0  
> 의존: PRD-13 (카테고리 트리), PRD-14 (관리자 v2)

---

## 배경

현재 Hugo Stack 테마를 사용 중이나, 테마의 제약 때문에 원하는 디자인을 자유롭게 구현하기 어렵다.
테마 의존성을 제거하고 `layouts/`에 완전한 커스텀 레이아웃을 구축한다.

### 기술 결정: Hugo 유지, Stack 테마 제거

Hugo(콘텐츠 처리 + GitHub Pages CI/CD)는 유지하되, Stack 테마를 제거하고  
`layouts/baseof.html`부터 모두 직접 작성한다.

| 항목         | 변경 전              | 변경 후                      |
|--------------|----------------------|------------------------------|
| 테마         | hugo-theme-stack     | 없음 (layouts/ 직접 구현)    |
| CSS          | Stack 테마 SCSS      | 커스텀 CSS (변수 + 클래스)   |
| 카테고리     | taxonomies/categories| Hugo 섹션 트리 (PRD-13)      |
| 프로필 정보  | hugo.yaml params     | data/profile.json            |

## 요구사항

### R1 — Hugo 기반 커스텀 레이아웃
- `themes/hugo-theme-stack` 의존성 제거 (`hugo.yaml` theme 항목 삭제)
- `layouts/baseof.html` — 전체 HTML 뼈대
- `layouts/home.html` — 홈 (소개 + 최근 글)
- `layouts/list.html` — 카테고리/섹션 글 목록
- `layouts/single.html` — 글 상세
- `layouts/404.html`
- `layouts/_partials/` — sidebar, head, scripts 등

### R2 — 데이터 분리
- `data/profile.json` — 이름, 소개글, 아바타 경로, 링크 목록
- `data/categories.json` — Composite 트리 (PRD-13 참조)

### R3 — 사이드바
- 프로필 영역 (아바타, 이름, 소개글)
- 링크 영역 (GitHub, 이메일 등 아이콘 + 링크)
- 카테고리 트리 (PRD-13 렌더링)
- 소개 페이지 링크

### R4 — 홈
- `data/profile.json`의 `bio` 렌더링
- 최근 글 6~8개 카드 목록

### R5 — 글 목록
- 직사각형 카드 (썸네일, 날짜, 태그, 제목, 요약)
- 날짜 내림차순 정렬
- 페이지네이션

### R6 — 글 상세
- 제목, 날짜, 읽기 시간, 카테고리/태그
- 마크다운 본문 (코드 하이라이트, 테이블, 인용구 등)
- 우측 sticky TOC (h2/h3, 스크롤 위치 추적)
- giscus 댓글·반응

### R7 — 다크모드
- CSS `@media (prefers-color-scheme: dark)` + 토글 버튼
- `localStorage`에 테마 저장

### R8 — 반응형
- 768px 미만: 사이드바 → 상단 햄버거 메뉴
- 1024px 미만: TOC 숨김
